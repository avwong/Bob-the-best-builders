353 | ## 🏗️ Architecture
354 | 
355 | ### Algorithm Choice: A* with Manhattan Distance
356 | 
357 | **Why A*?**
358 | - Optimal: Always finds the shortest path
359 | - Efficient: Uses heuristic to guide search
360 | - Well-tested: Industry standard for grid pathfinding
361 | 
362 | **Why Manhattan Distance?**
363 | - Perfect for 4-directional movement
364 | - Admissible heuristic (never overestimates)
365 | - Matches real supermarket navigation
366 | 
367 | ### Data Structures
368 | 
369 | **Priority Queue (Min-Heap)**
370 | - Efficiently manages open set
371 | - O(log n) insertion and extraction
372 | - Keeps nodes sorted by f-score
373 | 
374 | **Grid Representation**
375 | - 2D boolean array for memory efficiency
376 | - true = walkable, false = obstacle
377 | - Simple and fast to query
378 | 
379 | ### Performance Characteristics
380 | 
381 | - **Time Complexity**: O(b^d) where b is branching factor, d is depth
382 | - **Space Complexity**: O(b^d) for storing nodes
383 | - **Typical Performance**: < 100ms for 60×40 grid
384 | - **Large Grid (100×100)**: < 1 second
385 | 
386 | ## 🎯 Performance Tips
387 | 
388 | 1. **Reuse Grids**: Create grid once, reuse for multiple pathfinding calls
389 | 2. **Path Smoothing**: Only use when needed (adds processing time)
390 | 3. **Grid Size**: Keep grids reasonable (< 100×100 for real-time)
391 | 4. **Caching**: Cache frequently used paths
392 | 
393 | ```typescript
394 | // Good: Reuse grid
395 | const grid = createGridFromLayout(layout);
396 | const path1 = findPath(grid, start1, goal1);
397 | const path2 = findPath(grid, start2, goal2);
398 | 
399 | // Bad: Recreate grid each time
400 | const path1 = findPath(createGridFromLayout(layout), start1, goal1);
401 | const path2 = findPath(createGridFromLayout(layout), start2, goal2);
402 | ```
